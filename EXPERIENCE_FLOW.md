# OmidMed Platform - Experience Flow

## First User Experience

The first experience must create value before asking the clinic to register.

The user must not see login first.

Main journey:

1. User enters app.omidmed.com
2. User sees AI Product Preview experience
3. User uploads clinic logo
4. User selects product type
5. User selects bag color
6. User sees logo preview on selected bag
7. System gives simple design recommendation
8. User sees estimated price
9. User clicks submit order
10. User registers or logs in
11. Order is saved

---

## Main Hook

The first hook:

"لوگوی کلینیکت را آپلود کن و ببین روی کیف اختصاصی چطور دیده می‌شود."

## Page Sections

### 1. Hero Section

Goal:
Explain the value in 5 seconds.

Content:
- OmidMed Platform
- پنل هوشمند سفارش محصولات فیزیوتراپی
- آپلود لوگو
- انتخاب رنگ کیف
- پیش‌نمایش قبل از چاپ

Primary CTA:
شروع پیش‌نمایش رایگان

---

### 2. AI Product Preview Section

Inputs:

- Logo upload
- Bag color
- Product type
- Print color
- Quantity

Output:

- Bag mockup preview
- Logo placement
- Print readability score
- Recommended bag color
- Estimated price

---

### 3. Product Selection

Available MVP products:

- پک کامل بیمار
- کیف اسپان‌باند
- پد فیزیوتراپی
- ملحفه بیمار
- ژل
- کابل و الکترود

The system must be expandable for future products.

---

### 4. Price Preview

Before registration, show estimated price.

After registration, show final price and invoice.

---

### 5. Register After Value

Registration happens only after the user sees value.

Trigger:
User clicks "ثبت سفارش"

Then:
- Login
- Register
- Continue as clinic

---

## UX Rules

- Persian first
- RTL first
- Mobile first
- Maximum 3 steps before preview
- No technical words
- No complicated forms
- Large buttons
- Clear pricing
- Trustworthy medical design
- Fast experience

---

## AI Preview Rules

MVP:
Use mockup-based preview.

Future:
Use real AI image generation.

The system should later recommend:
- Best bag color
- Best print color
- Logo contrast quality
- Print readability
